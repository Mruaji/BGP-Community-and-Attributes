BGP Community Attribute

What is BGP Community?

A way to group destinations and apply routing policies

Attribute Type

Optional Transitive

Passed to other AS even if not recognized

Community Types

  • Standard Community: 32-bit (AS:Value)
    Example: 65000:100
  • Extended Community: 64-bit for MPLS VPNs
    Type:Administrator:Value
  • Large Community: 96-bit for 4-byte ASNs
    AS:Function:Parameter

Well-Known Communities

NO_EXPORT 65535:65281

Do not advertise to eBGP peers

NO_ADVERTISE 65535:65282

Do not advertise to any peer

INTERNET 0:0

Default community (all routes)

BGP Filtering: Prefix-List

Key Features

  • üîç Filter routes by prefix and length
  • ‚úÖ‚ùå Permit or deny specific networks
  • ‚¨áÔ∏è Sequential processing (top-down)
  • üö´ Implicit deny at end
  • üìè Supports "le" (less than or equal) and "ge" (greater than or equal) for prefix length ranges

Configuration Example

! Define prefix-list to filter routes
ip prefix-list FILTER seq 10 permit 10.0.0.0/8 le 24
ip prefix-list FILTER seq 20 deny 192.168.0.0/16
ip prefix-list FILTER seq 30 permit 0.0.0.0/0 le 32

! Apply to BGP neighbor
router bgp 65000
 neighbor 1.1.1.1 prefix-list FILTER in

Explanation

Line 10: Permit 10.0.0.0/8 and all subnets up to /24

Line 20: Deny all 192.168.0.0/16 networks

Line 30: Permit all other routes (0.0.0.0/0 le 32 = any prefix)

Application: Applied to inbound routes from neighbor 1.1.1.1

BGP Best Path Selection Algorithm

Evaluation Order (First Match Wins)

1
Weight

Highest weight (Cisco only, local to router)

2
Local Preference

Highest local preference (within AS)

3
Locally Originated

Prefer locally originated routes (network, aggregate, redistribute)

4
AS-PATH Length

Shortest AS-PATH length

5
ORIGIN Type

IGP > EGP > Incomplete

6
MED (Multi-Exit Discriminator)

Lowest MED (compared only for routes from same neighboring AS)

7
eBGP over iBGP

Prefer external BGP paths over internal BGP paths

8
IGP Metric to Next-Hop

Lowest IGP cost to reach BGP next-hop

9
BGP Router ID

Lowest BGP router ID (tie-breaker)

‚ö†Ô∏è Important: BGP evaluates these criteria in order. Once a tie is broken, no further evaluation occurs.

Summary & Best Practices

Key Takeaways

  • Community: Tag routes for policy (Optional Transitive)
  • Weight: Local preference, Cisco-only (0-65535, higher wins)
  • Local Preference: AS-wide outbound (0-4B, higher wins)
  • MED: Inbound traffic suggestion (0-4B, lower wins)
  • AS-PATH: Loop prevention (shorter preferred)
  • Prefix-List: Network filtering by prefix/length
  • Route-Map: Complex policy engine for matching and setting attributes

Best Practices

  • ‚úì Use Local Preference for outbound traffic within AS
  • ‚úì Use MED cautiously - neighbor's decision only
  • ‚úì Document all BGP policies and communities
  • ‚úì Test in lab before production deployment
  • ‚úì Remember implicit deny in ACLs/prefix-lists
  • ‚úì Understand path selection order thoroughly
  • ‚úì Use communities for scalable policy management
  • ‚úì Monitor BGP tables and routing changes regularly
